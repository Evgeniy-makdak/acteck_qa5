## Что это за советник

`Acteck.mq5` — аналитический советник/индикаторный инструмент для MetaTrader 5, который:

- анализирует исторические участки направленного движения;
- определяет развороты по заданному фильтру волатильности (в пунктах);
- строит трендовые отрезки на графике;
- рассчитывает вероятностные уровни продолжения/разворота;
- показывает рабочую таблицу по набору символов и фильтров;
- формирует CSV-отчёты в `MQL5/Files/Acteck/`.

Логика построена вокруг анализа трендовых участков, расчёта вероятностей разворота и оценки качества текущего сигнала по коррекционным ограничениям.

## Основные входные параметры

- `RepType` — авто/ручной выбор символа для отчётов.
- `Symb` — символ для ручного отчёта.
- `Timeframe` — ТФ отчёта.
- `StartDate / EndDate` — интервал исторического анализа.
- `UseCurrentEndDate` — если `true` (по умолчанию), конечная дата автоматически равна текущей дате.
- `MinPips` — базовый фильтр волатильности.
- `NameSet` — имя файла символов (без `.set`).
- `Level1..Level5` + `Timeframe1..Timeframe5` — столбцы таблицы.
- `Level4=0` и `Level5=0` по умолчанию (W1/MN выключены).  
  Чтобы включить их обратно, задайте `Level4` и/или `Level5` > 0.
- `Updater` — частота обновления, сек.
- `ATRPeriod` — период ATR для колонки ATR.
- `EnableAlerts` — звуковой сигнал при вероятности >= 60%.
- `ShowOnlyCurrentSymbol` — если `true` (по умолчанию), таблица показывает только 1 строку для текущего символа графика (быстрее и легче для терминала).

## Логика работы (коротко)

1. Советник загружает список инструментов из `MQL5/Files/Acteck/forex.set`.
2. Для каждого инструмента и каждого фильтра:
   - строит последовательность трендов;
   - оценивает вероятность по статистике прошлых движений;
   - считает коррекцию и текущее отклонение.
3. В таблице окрашивает ячейки:
   - синий — направление вверх (buy);
   - красный — направление вниз (sell);
   - белый — нейтрально.
4. По клику на ячейку:
   - переключает график на выбранный символ/таймфрейм;
   - рисует разметку и вероятностные уровни.
5. Пишет CSV-отчёты:
   - `MQL5/Files/Acteck/Table_1_<SYMBOL>.csv`
   - `MQL5/Files/Acteck/Table_2_<SYMBOL>.csv`

### Как считается вероятность

- `100%` — это максимальная историческая длина движения (по выбранному фильтру/ТФ).
- Значение в таблице рассчитывается от **старта текущей волны** до **последнего экстремума текущей волны** (достигнутого максимума сегмента).
- Это даёт вероятность "по достигнутому участку" для каждой ячейки фильтра/ТФ.
- Третий параметр в ячейке (`Тек. отклонение`) показывает отклонение текущей цены от последнего экстремума.
- Подпись `Current XX%` на графике остаётся динамической и относится к активной выбранной ячейке.

### Блокирующая подсветка в режиме `Вероятность`

Для ячеек формата `Вероятность % | Макс коррекция | Тек. отклонение` добавлен фильтр качества сигнала.

Ячейка подсвечивается красным как **недопустимый сигнал**, если превышен любой порог:

- `Макс коррекция` на всём трендовом участке >= `80%` фильтра волатильности;
- `Макс коррекция` от последнего экстремума >= `60%` фильтра волатильности;
- `Тек. отклонение` от последнего экстремума >= `40%` фильтра волатильности.

Пример для фильтра `50` пунктов: пороги `40/30/20`.  
Пример для фильтра `100` пунктов: пороги `80/60/40`.

В режиме `Вероятность` значения в ячейке означают:

- первое число — текущая вероятность;
- второе число — **максимальная коррекция на всём текущем трендовом участке**;
- третье число — **текущее отклонение цены от последнего экстремума**.

Цветовая логика:

- синий — сигнал `buy` созрел (вероятность >= 60 и лимиты не нарушены);
- оранжевый — сигнал `sell` созрел (вероятность >= 60 и лимиты не нарушены);
- красный — сигнал заблокирован (превышен один из лимитов 80/60/40).

## Debug-панель: расшифровка переменных

Верхняя строка `Диагностика` показывает служебные параметры активного символа/фильтра/ТФ.

### В режиме `Вероятность`

- `Pcur` — текущая вероятность (в %), рассчитанная по **текущей цене** внутри активной волны.
- `Dist` — текущая пройденная дистанция от старта активной волны до текущей цены (в points/пунктах котировки).
- `N` — размер исторической выборки в расчёте (это **не количество свечей**).  
  По факту `N` = число найденных исторических трендовых сегментов/волн в выбранном диапазоне `StartDate..EndDate` для конкретного символа, фильтра и ТФ.
- `idx` — ранговый индекс текущего движения в отсортированном по длине ряду волн (`0` = самая длинная волна).  
  Практически: количество исторических движений, длина которых **меньше или равна** `Dist`, вычисляется как `N - idx`.

Пояснение по вероятности:

- сравнение идёт по длинам волн в points;
- `Dist` сравнивается с историческими длинами;
- чем больше доля исторических волн, которые короче/равны текущему `Dist`, тем выше `Pcur`.

### В режиме `Длительность`

- `Tmax` — максимальная длительность исторической волны в выборке (в минутах).
- `Lmax` — максимальная длина исторической волны в выборке (в пунктах, отображается в старом формате MT4/10-point).

Смысл для принятия решений:

- `Tmax` показывает, как долго в истории в принципе тянулись подобные импульсы; если текущая волна уже близка к историческому потолку по времени, выше риск затухания/разворота.
- `Lmax` показывает исторический потолок хода по амплитуде; полезно как ориентир, где движение статистически «перегрето».
- В ячейках в этом режиме выводится `% длительности от средней`:  
  `Duration% = (Tcur / Tavg) * 100`, где:
  - `Tcur` — длительность текущей волны,
  - `Tavg` — средняя длительность исторических волн.
- Применение: помогает не входить поздно в «уставшее» движение, когда по времени волна уже заметно выше среднего.

### В режиме `Скорость`

- `Vcur` — текущая скорость активной волны по формуле `L/T` (пункты в минуту).
- `Vavg` — средняя историческая скорость по выборке (пункты в минуту).

Смысл для принятия решений:

- `Vcur` показывает, насколько быстро рынок проходит дистанцию **прямо сейчас**.
- `Vavg` — базовый исторический темп для данного инструмента/фильтра/ТФ.
- В ячейках в этом режиме выводится `% скорости от средней`:  
  `Speed% = (Vcur / Vavg) * 100`.
- Интерпретация:
  - сильно выше `100%` — импульсный/ускоренный рынок (можно раньше переходить в сопровождение прибыли, жестче контролировать откат);
  - около `100%` — режим близок к статистической норме;
  - заметно ниже `100%` — вялое движение, выше риск «пилы» и ложных протяжек.

Практическое преимущество `Скорости`:

- отделяет «длинную, но медленную» волну от «короткой, но очень быстрой»;
- помогает фильтровать входы по качеству импульса, а не только по факту достигнутой длины/длительности;
- полезен для адаптации тактики сопровождения (частичная фиксация, трейлинг, ожидание добоя до цели).

Кнопка `[x]/[ ] Debug` включает и отключает отображение этой строки без перезапуска советника.

## Примеры интерпретации (для новичка)

Ниже простые «рабочие» примеры, как читать значения на панели и в таблице.

### 1) Пример для режима `Вероятность`

Допустим, на панели:

- `Pcur=41%`
- `Dist=1996`
- `N=784`
- `idx=464`

Что это значит:

- рассмотрено `784` исторических волн;
- текущая волна прошла расстояние `Dist=1996` points от старта;
- `idx=464` означает, что в отсортированном ряду текущая дистанция находится на позиции 464;
- волн, которые **длиннее** текущей дистанции: примерно `464`;
- волн, которые **короче или равны** текущей дистанции: `784 - 464 = 320`.

Практический вывод:

- при `Pcur` заметно ниже 60% движение статистически еще не в «зоне зрелости»;
- это не сигнал «обязательно входить», но аргумент, что потенциал продолжения еще может быть;
- решение принимается только вместе с контекстом (направление тренда, коррекция, новости, риск).

### 2) Пример для режима `Длительность`

Допустим:

- в ячейке `Duration%=210%`
- на панели `Tmax=48960m`, `Lmax=919p`

Как считается:

- `Duration% = (Tcur / Tavg) * 100`;
- `210%` означает, что текущая волна идет в 2.1 раза дольше средней исторической.

Практический вывод:

- волна уже «долго живет» относительно нормы;
- повышается вероятность замедления/пилы/разворотной фазы;
- для торговли это повод:
  - осторожнее с новыми входами «вдогонку»,
  - строже сопровождать уже открытую позицию (частичная фиксация, подтяжка стопа),
  - ждать подтверждения перед добавлением объема.

### 3) Пример для режима `Скорость`

Допустим:

- в ячейке `Speed%=165%`
- на панели `Vcur=0.082`, `Vavg=0.050` (пункты в минуту)

Как считается:

- скорость волны: `V = L / T`;
- `Speed% = (Vcur / Vavg) * 100 = 165%`.

Практический вывод:

- рынок движется быстрее своей исторической «нормы»;
- это признак импульсной фазы: цель может достигаться быстрее, но и откаты часто резче;
- для торговли это обычно означает:
  - не опаздывать с управлением риском,
  - учитывать повышенную волатильность,
  - избегать завышенного лота «на эмоциях ускорения».

И наоборот: если `Speed%` заметно ниже 100% (например 60-70%), рынок вялый; чаще полезно снижать ожидания по быстрому добою цели и избегать агрессивных входов.

## Динамический Target после возможного разворота

Начиная с текущей версии, в режиме `Вероятность` используется **динамический target** (если хватает исторических данных), а не только статический ориентир.

### Что именно считается

Идея: если текущая волна пришла к вероятности `Pcur`, то смотрим в истории похожие ситуации и анализируем, какой длины была **следующая волна после разворота**.

То есть target строится не «из головы», а из статистики реальных последующих движений.

### Математическая модель

Для текущего состояния (`symbol + timeframe + filter`):

1. Считаем текущую вероятность `Pcur`.
2. Берём исторические волны из того же диапазона данных.
3. Отбираем волны, где вероятность близка к текущей:  
   `|P_i - Pcur| <= band`, где сейчас `band = 10` (%-пунктов).
4. Для каждой такой волны берём **следующую волну** (после разворота):
   - её длину `L_next` (в points),
   - её длительность `T_next` (в минутах).
5. Считаем средние:
   - `TargetPoints = mean(L_next)`
   - `TargetBars = mean(T_next) / TF_minutes`
6. Проецируем цену target в противоположную сторону:
   - если текущая волна вверх -> target вниз от текущей цены;
   - если текущая волна вниз -> target вверх от текущей цены.

### Что выводится на графике и в debug

В режиме `Вероятность`:

- на графике линия `Target dyn ...p` — динамический целевой уровень;
- в debug-строке:
  - `Tdyn` — средняя длина target в пунктах (MT4-style /10-point),
  - `bars` — тот же target, пересчитанный в бары текущего ТФ,
  - `Ns` — число исторических случаев, попавших в выборку.

Если исторических совпадений недостаточно, система автоматически использует статический fallback (`Target static`).

### Простой числовой пример

Пусть в истории, когда вероятность была близка к текущей, следующие волны после разворота имели длины:

- 20, 30, 40, 50, 60 баров.

Тогда:

- `TargetBars = (20 + 30 + 40 + 50 + 60) / 5 = 40` баров.

Это и есть ожидаемая средняя «протяжка» следующей волны после разворота на текущем уровне вероятности.

### Практический смысл для торговли

- target становится адаптивным к текущему режиму рынка;
- снижается зависимость от фиксированного hardcoded-уровня;
- проще планировать сопровождение:
  - частичную фиксацию,
  - перенос стопа,
  - оценку «хватает ли места» до статистической цели.

Важно: это статистический ориентир, а не гарантированный исход. Использовать вместе с риск-менеджментом и рыночным контекстом.

## Рекомендации по торговому применению

Инструмент используется как аналитический модуль принятия решений:

- сначала фильтрация пар по вероятности;
- затем сверка уровней, коррекции и текущего отклонения;
- вход в сделку подтверждается рыночным контекстом (уровни/новости/ATR).

Согласно исходной методологии Evgeniy Acteck:

- не входить в рынок без сигнала;
- работать от ложного пробоя и разворотной логики;
- соблюдать риск-менеджмент и целевое соотношение риска к прибыли.
